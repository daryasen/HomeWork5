# ZooERP - Система управления зоопарком

## Описание проекта
ERP-система для автоматизации учета животных и инвентаря в Московском зоопарке.

## Функциональность
- Учет животных и их характеристик
- Проверка здоровья животных перед приемом
- Расчет общего количества потребляемой еды
- Формирование списка животных для контактного зоопарка
- Инвентаризация имущества

## Примененные принципы SOLID

### SRP (Single Responsibility Principle)
- Каждый класс имеет одну ответственность:
  - ZooService - управление зоопарком
  - VeterinaryClinic - проверка здоровья животных
  - Каждый тип животного - собственная логика проверки здоровья

### OCP (Open-Closed Principle)
- Легко добавлять новые типы животных, наследуясь от Animal, Herbivore или Predator
- Новые типы инвентаря наследуются от Thing

### LSP (Liskov Substitution Principle)
- Все животные могут быть использованы вместо базового класса Animal
- Все травоядные могут быть использованы как Herbivore

### ISP (Interface Segregation Principle)
- Интерфейсы разделены по функциональности:
  - IAlive - для живых существ
  - IInventory - для инвентаризации
  - IHealthCheckable - для проверки здоровья

### DIP (Dependency Inversion Principle)
- Зависимости инжектируются через конструкторы
- Использование DI-контейнера для управления зависимостями

## Использование DI-контейнера
В проекте используется Microsoft.Extensions.DependencyInjection:
- Регистрация сервисов в ContainerConfig.Configure()
- Внедрение зависимостей через конструктор ZooService
